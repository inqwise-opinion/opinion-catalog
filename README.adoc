= Opinion Catalog

Opinion Catalog provides a centralized, authoritative catalog of shared, well-defined items used across the Opinion platform.
It exposes stable, read-oriented APIs for commonly used lists such as geographic information (countries, cities, timezones) and product-related items (service packages, plans, tiers).

== Modules

* `opinion-catalog-common` – shared DTOs and helpers used across services consuming catalog data.
* `opinion-catalog` – HTTP service that exposes the catalog API.

== Requirements

* Java 21 (Temurin 21 or newer JDK).
* Maven 3.9+ installed locally.

== Add as a Dependency

To use the shared catalog types:

[source,xml]
----
<dependency>
    <groupId>com.inqwise.opinion</groupId>
    <artifactId>opinion-catalog-common</artifactId>
    <version>${revision}</version>
</dependency>
----

To depend on the HTTP catalog service client or embed the service, use:

[source,xml]
----
<dependency>
    <groupId>com.inqwise.opinion</groupId>
    <artifactId>opinion-catalog</artifactId>
    <version>${revision}</version>
</dependency>
----

Replace `${revision}` with the released version you wish to consume
(or `1.0.0-SNAPSHOT` when consuming snapshots from Sonatype).

== Development

[subs="+quotes"]
----
# Build the whole multi-module project #
./mvnw clean verify

# Build only the shared common library #
./mvnw -pl opinion-catalog-common clean verify

# Build and test the HTTP catalog service #
./mvnw -pl opinion-catalog clean verify
----
